## 1. 專案概述

### 1.1 背景與動機

現代人生活忙碌，常常在網路上看到喜歡的食譜卻沒有一個統一的地方可以收藏與管理。市面上雖有許多食譜網站，但多數功能繁雜、廣告干擾多，缺乏針對「個人收藏與快速查找」的簡潔工具。

**食譜收藏夾系統**旨在提供一個乾淨、直覺的平台，讓使用者能輕鬆新增、搜尋、瀏覽食譜，並獲得個人化的推薦，同時清楚呈現每道料理所需的食材與步驟。

### 1.2 目標用戶

| 角色 | 說明 |
|------|------|
| **管理員（Admin）** | 負責管理系統內容，可新增、編輯、刪除所有食譜，管理使用者帳號與系統設定 |
| **使用者（User）** | 一般註冊會員，可瀏覽、搜尋、收藏食譜，查看食材清單與料理步驟，獲得推薦食譜 |

### 1.3 核心價值主張

- **簡潔直覺**：提供乾淨的介面，讓使用者專注於食譜內容本身
- **快速查找**：強大的搜尋功能，透過關鍵字、食材、分類等多維度搜尋食譜
- **智慧推薦**：根據使用者的收藏偏好與瀏覽紀錄，推薦可能感興趣的食譜
- **完整資訊**：每道食譜皆包含詳細的食材清單與分步驟說明，確保使用者能順利完成料理

## 2. 功能需求

### 2.1 新增食譜

> **使用者故事：**  
> 作為**管理員/使用者**，我希望能**新增一道食譜（包含名稱、描述、圖片、食材、步驟、分類標籤）**，以便**將自己喜歡的料理記錄到系統中方便日後查閱**。

**功能細節：**
- 填寫食譜名稱、簡介描述
- 上傳食譜封面圖片
- 設定分類標籤（如：中式、西式、日式、甜點、素食等）
- 設定料理難度（簡單 / 中等 / 困難）
- 設定預估烹飪時間
- 新增食材清單（食材名稱、數量、單位）
- 新增烹飪步驟（步驟編號、說明文字）

### 2.2 搜尋食譜

> **使用者故事：**  
> 作為**使用者**，我希望能**透過關鍵字、食材名稱或分類標籤來搜尋食譜**，以便**快速找到想要的料理**。

**功能細節：**
- 關鍵字搜尋（模糊比對食譜名稱與描述）
- 依分類標籤篩選
- 依難度篩選
- 依烹飪時間篩選
- 搜尋結果排序（最新、最多收藏、烹飪時間）

### 2.3 推薦食譜

> **使用者故事：**  
> 作為**使用者**，我希望系統能**根據我的收藏紀錄與瀏覽偏好推薦相關食譜**，以便**發現更多我可能喜歡的料理**。

**功能細節：**
- 根據使用者收藏的食譜分類，推薦同類型的熱門食譜
- 首頁顯示「今日推薦」區塊
- 顯示「其他人也收藏了」的相關食譜
- 根據季節或節日推薦當季食譜

### 2.4 列出食材

> **使用者故事：**  
> 作為**使用者**，我希望能**查看每道食譜完整的食材清單，並可匯出為購物清單**，以便**在購買食材時不會遺漏**。

**功能細節：**
- 食譜詳情頁清楚列出所有食材、份量與單位
- 支援依人數調整食材份量（如：原 2 人份 → 調整為 4 人份）
- 收藏多道食譜後，可合併產生食材購物清單
- 購物清單可勾選已購買項目

### 2.5 烹飪步驟

> **使用者故事：**  
> 作為**使用者**，我希望能**按順序查看每道食譜的詳細烹飪步驟**，以便**依照指引一步步完成料理**。

**功能細節：**
- 步驟以編號方式清楚排列
- 每一步驟可包含文字說明
- 支援步驟間的「上一步 / 下一步」導覽
- 烹飪模式：大字體顯示，方便在廚房邊看邊做

### 2.6 使用者帳號管理

> **使用者故事：**  
> 作為**使用者**，我希望能**註冊帳號並登入系統**，以便**擁有個人的收藏夾與使用紀錄**。

**功能細節：**
- 使用者註冊（帳號、密碼、暱稱）
- 使用者登入 / 登出
- 個人資料編輯
- 查看個人收藏的食譜清單
- 查看個人新增的食譜清單

### 2.7 食譜收藏功能

> **使用者故事：**  
> 作為**使用者**，我希望能**將喜歡的食譜加入收藏夾**，以便**日後快速找到並重複使用**。

**功能細節：**
- 一鍵收藏 / 取消收藏
- 個人收藏夾頁面，列出所有已收藏食譜
- 收藏夾內支援搜尋與篩選

### 2.8 管理員後台

> **使用者故事：**  
> 作為**管理員**，我希望能**管理所有食譜內容與使用者帳號**，以便**維護系統內容品質與正常運作**。

**功能細節：**
- 檢視、編輯、刪除任意食譜
- 檢視、停用使用者帳號
- 食譜審核機制（可選）
- 系統數據統計（食譜總數、使用者總數、熱門食譜等）

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 說明 |
|------|------|
| 後端框架 | Flask（Python） |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite |
| 前端 | HTML + CSS + JavaScript（原生） |
| 部署環境 | 本機開發環境 |

### 3.2 效能考量

- 搜尋回應時間應在 1 秒內完成
- 頁面載入時間不超過 3 秒
- 圖片上傳限制：單張不超過 5MB
- 資料庫查詢應使用適當索引以優化效能

### 3.3 安全考量

- 使用者密碼需以雜湊（hash）方式儲存，禁止明文儲存
- 所有表單需防止 SQL Injection 與 XSS 攻擊
- 管理員操作需進行權限驗證
- 使用者只能編輯 / 刪除自己新增的食譜

### 3.4 使用者體驗

- 響應式設計（RWD），支援手機與桌面瀏覽器
- 介面語言：繁體中文
- 操作流程簡潔直覺，新手也能輕鬆上手
